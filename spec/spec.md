# 飞机游戏项目优化规范

## 项目概述

对 flyfly 飞机射击游戏项目进行代码优化，解决重复代码、未使用变量、性能问题和代码结构问题。

## 优化目标

1. **代码质量提升** - 消除重复代码，删除未使用变量
2. **性能优化** - 减少每帧不必要的计算和查找
3. **代码结构改进** - 修复缩进错误，拆分过长函数
4. **错误处理增强** - 添加必要的空值检查

## 优化范围

### 高优先级（必须修复）

| 问题 | 文件 | 描述 |
|------|------|------|
| 重复函数定义 | player.gd | `set_base_attribute_level` 函数定义两次 |
| 缩进错误 | bullet.gd | 第200-203行缩进不正确 |
| 缺失字典键 | enemy.gd | `skill_rarity` 缺少 `legendary` 键 |
| 重复销毁逻辑 | bullet.gd | 敌人子弹同时调用 queue_free 和对象池返回 |

### 中优先级（代码质量）

| 问题 | 文件 | 描述 |
|------|------|------|
| 重复输入检测 | player_movement.gd | `_handle_movement` 和 `_handle_rotation` 重复输入检测 |
| 重复对象池逻辑 | bullet.gd | 对象池返回逻辑重复4处 |
| 未使用变量 | enemy.gd | 约15个未使用的变量 |
| 重复技能图标函数 | game.gd | `_update_skill_icons_display` 和 `update_skill_icons_display` 重复 |
| 硬编码值 | 多个文件 | 基础速度、地图尺寸等硬编码 |

### 低优先级（性能优化）

| 问题 | 文件 | 描述 |
|------|------|------|
| 每帧遍历敌人 | player.gd | `_check_enemy_collision` 每帧遍历所有敌人 |
| 每帧查找玩家 | enemy.gd | `_physics_process` 中每帧查找玩家 |
| 重复碰撞检测 | bullet.gd | 同时使用信号和手动检测 |
| 过长函数 | enemy.gd | `_spawn_skill_drop` 超过160行 |

## 技术约束

1. 保持现有功能不变
2. 不改变游戏逻辑
3. 保持向后兼容性
4. 遵循 GDScript 代码规范

## 验收标准

1. 所有重复代码已消除
2. 所有未使用变量已删除
3. 所有缩进错误已修复
4. 所有硬编码值已提取为常量或配置
5. 项目可正常运行，无报错
